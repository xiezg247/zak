Python Skills 通过 Services 暴露四大业务能力给 AI

现状：只有 1 个 Python Skill，4 个工具

skills/vnpy_context_skill.py（当前）
VnpyContextSkill ├─ get_quote_context() ← 获取选中标的上下文 ├─ get_watchlist() ← 获取自选池 ├─ get_bars_summary() ← 查询K线概览 └─ get_backtest_summary() ← 获取最近回测摘要 问题： 1. 一个 Skill 做四件事，职责不清（获取上下文 ≠ 回测 ≠ K线查询） 2. 直接 import bar_store/bars/session_context，紧耦合 3. 没有选股工具 4. 没有"触发回测"能力，只能"看结果" 5. 工具参数映射到业务逻辑的翻译逻辑混在 Skill 里

按业务域拆分的 Skill 设计详情

VnpyContextSkill

（简化版）

  • get_quote_context() → 当前选中标的 + 页面
  • get_page_state() → 当前在哪个页面
  • 职责：仅提供"用户在看什么"的上下文

VnpyDataSkill

（新增）

  • get_bars_summary(symbol, scope)
  • get_bars_data(symbol, scope, days) → 返回 OHLCV 数据
  • get_quote(symbol) → 实时行情
  • get_market_overview() → 市场概览（涨幅榜等）
  • 依赖：BarService + QuoteService

VnpyBacktestSkill

（新增）

  • list_strategies() → 可用策略列表
  • run_backtest(strategy, symbol, ...) → 触发回测
  • get_backtest_result() → 最近一次回测结果
  • compare_strategies(...) → 批量回测对比
  • 依赖：BacktestService

VnpyScreeningSkill

（新增）

  • list_screeners() → 可用选股条件
  • screen_by_condition(name, top_n) → 选股
  • screen_custom(conditions) → 组合条件选股
  • 依赖：ScreeningService

VnpyWatchlistSkill

（新增）

  • get_watchlist() → 自选池列表（从 VnpyContextSkill 移过来）
  • add_to_watchlist(symbol) → 加入自选
  • remove_from_watchlist(symbol) → 移出自选
  • get_watchlist_quotes() → 自选池实时行情
  • 依赖：WatchlistService + QuoteService

Skill 如何获取 Service 引用 — 三种方案

A

全局 AppContext 单例

在 AshareEngine.init() 时向全局 AppContext 注册 Services，Skill 通过 get_app_context().bar_service 获取

⚠️ 最简单但引入隐式全局状态
B

SkillEngine 构造时注入

SkillEngine(services={...}) 在 LlmEngine 初始化时注入 Service 字典，Skill.setup() 时传入

✅ 显式依赖，可测试
C

Skill 构造时注入

每个 Skill 的 __init__ 接受 Service 参数，SkillEngine.load_python_skills 时实例化传入

最显式但需要改动 SkillEngine 的类加载逻辑
推荐方案 B：SkillEngine 构造时注入 Service 字典。

SkillEngine(services={"bar": bar_service, "backtest": backtest_service, ...})

然后在 SkillTemplate.setup() 时将 self._services 传递给实例。
好处：不破坏现有 SkillTemplate 抽象，不引入全局状态，方便 mock 测试。
