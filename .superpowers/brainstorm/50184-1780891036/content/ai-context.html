设计第三章：Agent Skills 知识注入 + AI 上下文优化

让 LLM 更懂 A 股业务，上下文传递更高效

现状梳理：两种 Skill 形态的职责

当前分工
Agent Skills (SKILL.md) Python Skills (SkillTemplate) ├─ tickflow/SKILL.md ├─ VnpyContextSkill │ ├─ 行情/K线/财务查询示例 │ └─ 4 个工具函数 │ └─ 知识型：教 LLM 怎么用 TickFlow │ │ ├─ （计划新增 4 个 Skill） └─ tushare-data/SKILL.md │ └─ 16+ 工具函数 ├─ 10 种数据工作流模板 │ └─ references/数据接口.md └─ 工具型：注册 Function Calling 工具

问题与优化

问题 1：Agent Skills 知识注入冗余 — 每次对话都塞完整 SKILL.md

当前 SkillEngine.build_skills_prompt()
遍历全部 Agent Skills → 拼接每个 SKILL.md 的 body + references 列表 + scripts 列表 → 大段文本注入 System Prompt（tushare-data 尤其大） 问题：Token 浪费，大部分内容与当前问题无关

优化：分层知识注入

改造后 System Prompt 结构
1. 核心系统提示词（不变，约 500 tokens） 2. 可用工具摘要（自动生成，约 200 tokens） "你拥有：数据查询(K线/行情/财务)、策略回测、选股筛选、自选管理" 3. Agent Skills 知识摘要（仅 name+description，约 100 tokens/Skill） "tickflow: 行情/K线说明，需要时用 read_skill_file 查阅" 4. 策略知识摘要（从 registry.py 提取，约 300 tokens） "双均线策略：适用趋势市，不适用震荡市。参数：快线5/慢线20" 5. 当前上下文（选中标的、页面状态，约 100 tokens） 总计约 1500 tokens，比之前塞完整 SKILL.md 大幅减少

问题 2：AI 上下文事件链路过长

当前链路
QuotesPage 选股 → build_quote_context() 组装 → event_engine.put(EVENT_AI_CONTEXT) 事件广播 → LlmEngine._on_context_event() 回调 → set_ai_context(data) 全局变量 + 锁 → signals.context_changed.emit() UI 通知 → Skill 调用 get_ai_context() 从全局读 问题：事件→回调→全局变量→锁→信号，链路过长且不必要

优化：Lazy fetch，去掉事件广播

改造后：QuoteService 持有上下文
QuotesPage 选股 → quote_service.set_current_selection(...) ← 一行 setter Skill 调用 → engine.quote_service.get_current_context() ← Lazy 读取 去掉：EVENT_AI_CONTEXT 事件、session_context 全局变量、线程锁 保留：Qt signals 通知 UI（已有机制）

问题 3：策略知识未暴露给 AI

strategies/registry.py 有 metadata，但 LLM 不知道
StrategyMeta(name="双均线策略", suitable="趋势明显...", unsuitable="横盘震荡...") 这个信息只在回测页标签里显示，AI 对话完全不知道！ 优化：在 System Prompt 中注入策略知识摘要（从 registry.py 自动提取）
