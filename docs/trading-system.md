# 交易体系需求说明

> **阶段**：框架期需求文档。定义「短线为主、中线为辅、超短线玩法」下的产品能力边界与演进路线；**非**实盘下单系统，定位为个人 A 股现货**投研 + 规则辅助**终端。
>
> 技术实现见 [产品说明](./product-plan.md)、[架构说明](./architecture.md)；本文聚焦**交易逻辑 → 功能映射**。

## 1. 产品定位

### 1.1 交易风格分层

| 层级 | 持股周期 | 核心收益 | 在 zak 中的权重 |
|------|----------|----------|-----------------|
| **极致短线（主）** | 1–3 交易日 | 情绪溢价、资金价差、T+1 流动性红利 | **P0**：择时、人气选股、买卖点规则、隔日卖点、仓位系数、风控闸 |
| **短线波段（辅）** | 3–10 交易日 | 题材发酵、龙头趋势延续 | **P1**：雷达展望卡、双均线/突破策略、持仓跟踪 |
| **中线观察（辅）** | 2–8 周 | 趋势 + 基本面共振 | **P2**：团队分析、财务/风险 Agent、回测验证 |

### 1.2 底层公式（系统须内化，不对用户承诺收益）

```text
总收益 ≈ (胜率 × 盈亏比 − 1) × 有效交易频率 − 摩擦成本
```

| 参数 | 极致短线合理区间 | 系统表达 |
|------|------------------|----------|
| 胜率 | 40%–55% | 回测 / 交易流水统计（未来） |
| 盈亏比 | ≥ 2:1 | 单笔止损 −2%、止盈分批规则（未来风控模块） |
| 有效频率 | 随情绪周期变化 | 择时闸限制「可做 days」 |
| 摩擦成本 | 佣金 + 滑点 | 回测 `AShareTemplate` 已含 T+1、整手 |

**设计原则**：靠盈亏比取胜，靠风控存活；系统输出**规则参考**与**环境提示**，不构成买卖建议（沿用 `SIGNAL_DISCLAIMER`）。

### 1.3 七大模块 → 功能域总览

```text
┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐
│  择时   │ → │  选股   │ → │  买卖   │ → │  仓位   │ → │  风控   │ → │  复盘   │ → │  纪律   │
└────┬────┘   └────┬────┘   └────┬────┘   └────┬────┘   └────┬────┘   └────┬────┘   └────┬────┘
     │             │             │             │             │             │             │
  市场环境      选股 Hub       策略信号       持仓区        风控闸        笔记/流水      交易计划
  情绪周期      雷达/市场      买卖点规则     仓位系数      回撤熔断      运行历史       计划内交易
  板块资金      共振标的       分 K 辅助      分组 Tab       单日止损      AI 解读        违规标记
```

### 1.4 首屏：个人交易体系 Playbook

侧栏 **Ctrl+1 · 交易体系** 为 zak 默认首屏（非行情页、非工作台导航）。定位为**每天开盘前必读的操作宪法**：规则全文 + 今日状态对照，不承担页面路由。

| 区块 | 内容 | 数据来源 |
|------|------|----------|
| 对照条 | 情绪 / 风控 / 日盈亏 / 今日计划 / 持仓 / 纪律进度 | 本地 SQLite + QSettings + 内存缓存 peek，**首屏 activate 不拉 TickFlow** |
| §1 择时 | 做不做 | 用户 Markdown + 模板 seed |
| §2 选股 | 做什么 | 用户 Markdown + Profile / 硬过滤**只读镜像** |
| §3 买卖 | 怎么做 | 用户 Markdown |
| §4 仓位与风控 | 风控铁则 | 用户 Markdown + `TradingRiskPrefs` **只读镜像** |
| §5 纪律 | 每日 checklist + 规则正文 | 5 项默认检查（按交易日重置）；计划外持仓标红 |

**三层内容模型**

```text
L1 模板层 — 按 Strategy Profile seed（ultra_short / short_swing …）
L2 用户层 — 编辑过的 Markdown 持久化于 trading_playbook_sections，优先于模板
L3 镜像层 — §2 / §4 底部自动附加当前 Profile、硬过滤、风控参数（只读）
```

**Profile 切换**：已编辑章节不覆盖；仍为旧 Profile 默认模板的章节，切换时可确认套用新 Profile 对应章节。

**AI**：§5「一句纪律」与悬浮球快捷动作「今日一句纪律」需**用户主动点击**；注入 Playbook 摘要与今日对照，不自动发送。

**实现落点**：`ui/home/`、`services/trading_playbook.py`、`config/playbook_templates/`。

## 2. 择时体系：情绪周期决定「做不做」

极致短线最大亏损来源是**退潮期强行操作**。zak 须把「环境 → 仓位上限 → 允许的模式」做成可计算、可展示、可注入 AI 的一等公民。

### 2.1 情绪周期定义

| 阶段 | 判定条件（可量化） | 策略 | 建议总仓位 |
|------|-------------------|------|------------|
| **冰点** | 连板高度 ≤ 2；跌停 > 15；涨跌比偏空 | 空仓或 ≤ 1 成试错 | 0%–10% |
| **启动** | 出现 ≥ 3 板龙头；涨停 > 50；题材有梯队 | 试错首板/二板 | 30%–50% |
| **发酵/高潮** | 连板梯队完整；龙头加速；板块批量涨停 | 龙头/前排跟风 | 60%–80% |
| **分歧** | 龙头炸板；涨跌停接近；跟风掉队 | 减仓；仅核心低吸 | ≤ 30% |
| **退潮** | 龙头跌停；连板批量断板；亏钱效应扩散 | **无条件空仓** | 0% |

> **读表说明**：表中「龙头 / 连板 / 梯队」指**全市场短线情绪代理指标**，用于回答「今天做不做、总仓位上限、允许哪种模式」；**不**等同于个股推荐，也不要求个人必须跟随某一板块龙一。具体标的筛选见 [§3 选股体系](#3-选股体系只做有辨识度的人气股)。

> **实现注记**：连板高度、涨停/跌停家数可从 `market_breadth`、Tushare `limit_list_d`、市场页 `limit_up` / **连板榜**（`rank_id=limit_times`）聚合；情绪周期与雷达 D-01 已消费 `limit_times` 分布。

### 2.2 辅助验证指标

| 指标 | 阈值 / 规则 | 系统动作 |
|------|-------------|----------|
| 两市成交额 | < 1 万亿 | 降低仓位系数；选股 Hub 展示警告 |
| 大盘 5 日线 | 拐头向下 | 减少操作频率提示；雷达/选股默认保守硬过滤 |
| 监管异动 | 接近 10 日 4 次异动 / 30 日 200% | 概览「监管异动」提醒 + AI `assess_regulatory_deviation` **已有** |
| 恐贪指数 | 已有 `vnpy-sentiment` | 作为 `sentiment_gate` 维度权重调制（已实现） |

### 2.3 功能需求

| ID | 能力 | 状态 | 落点 |
|----|------|------|------|
| T-01 | 市场广度条：涨跌停、涨跌家数、成交额 | **已有** | 市场页 `stats_bar` + `market_breadth` |

## 4. 策略与买卖点：量化规则，拒绝主观

极致短线的买卖点须**可执行、可复制**。zak 以「策略插件 + 规则参数 + 分 K 辅助」表达，而非主观文案。

### 4.1 三类买点（策略模式）

| 模式 | 适用环境 | 量化规则（输入） | 输出 |
|------|----------|------------------|------|
| **打板** | 情绪上升期；龙头/核心跟风 | 涨停价触及；封板回封；时间 ≤ 10:30；非一字缩量 | `buy` + 涨停价参考 |
| **半路** | 题材发酵；带量拉升 | 涨幅 3%–7%；突破 key level；时段 9:40–10:30 | `buy` + 突破价参考 |
| **低吸** | 核心分歧；情绪未坏 | 回调至 MA5 或日内 −3%~−5% 承接；优先 14:30 后 | `buy` + 低吸价参考 |

| 模式 | 默认策略插件 | 依赖数据 |
|------|-------------|----------|
| 打板 | `AshareLimitBoardStrategy` | **已有** | 日 K 回测；信号/选股优先本地 1m；分 K 回测见 `AshareLimitBoardMinuteStrategy` |
| 半路 | `AshareIntradayBreakoutStrategy` | **已有** | 日 K 代理；分 K 见 `AshareIntradayBreakoutMinuteStrategy` |
| 低吸 | `AsharePullbackStrategy` | **已有** | 日 K MA5 回踩；分 K 见 `AsharePullbackMinuteStrategy` |
| 隔日卖点 | `AshareOvernightExitStrategy` | **已有** | 持仓 overlay；分 K 回测见 `AshareOvernightExitMinuteStrategy` |
| 短线波段（已有） | `AshareShortBreakoutStrategy` | 日 K 突破 + 量比 |

> **现状**：自选信号区默认 `AshareShortBreakoutStrategy`（短线放量突破）。`AshareLimitBoardStrategy` 等极致短线策略通过 Profile 切换为 `ultra_short` 启用（见 [策略配置方案](./strategy-profiles.md)）；`ultra_short` Profile 已绑定隔日退出 overlay + `evaluate_overnight_exit` AI 工具。隔日规则引擎由 `overnight_exit_rules` 统一 quote/分 K 路径；持仓区展示 `exit_rules` 列（触发/临近/未触发）。雷达「未来·展望」按策略 key 隔离缓存，切换策略会强制重算。

**工程（2026-06）**：自选/持仓信号磁盘缓存与雷达展望缓存迁至 `storage/cache/`；app_db 与各 store 复用 `sqlite_cache_session`；`vnpy_llm` 在 launcher / 主窗口懒加载，业务跳转经 `vnpy_common` LLM 桥接（见 §9）；策略 whitelist 与分 K 模式映射收敛至 `strategies/registry.py`。

### 4.2 隔日卖点铁则（规则引擎）

| 类型 | 条件 | 动作 |
|------|------|------|
| 止盈 | 开盘冲高 3%–5% 且量能不足 | 分批止盈提示 |
| 止盈 | 主板涨停 | 持有；炸板回封无力 → 卖 |
| 止盈 | 20cm 冲高 ≥ 10% 滞涨 | 止盈 |
| 止损 | 该高开却低开，30 分钟不翻红 | 无条件止损 |
| 止损 | 日内浮亏 ≤ −5% | 立即止损 |
| 止损 | 跌破开盘价 / 昨收且反弹无力 | 卖 |
| 时间 | 极致短线 | 优先上午完成卖出；弱势下午不格局 |

**表达形式**：`ExitRuleSnapshot` — 每条持仓展示「触发/临近/未触发」标签（扩展现有 `PositionSnapshot.exit_signal`）。

## 5. 仓位管理：与风险匹配

zak **不连接实盘下单**，仓位模块表达「计划仓位 vs 记账仓位」。

### 5.1 规则

| 规则 | 参数 |
|------|------|
| 单票上限 | 新手 ≤ 2 成；成熟核心龙头 ≤ 5 成 |
| 总仓位 | 永不满仓；随情绪周期系数（§2.1） |
| 分散 | 3–4 只 |
| 加仓 | **仅盈利头寸可加仓**；亏损绝不补仓 |

### 5.2 功能需求

| ID | 能力 | 状态 | 说明 |

| P-02 | T+1 锁定提示 | **已有** | `position_t1_locked` |

## 6. 风控体系：活下去是第一目标

### 6.1 三层风控

| 层级 | 阈值 | 动作 |
|------|------|------|
| 单笔 | 最大亏损 ≤ 总资金 2% | 登记持仓时反推最大可承受跌幅 |
| 单日 | 单日亏损 ≥ 3% | toast 建议减频新开仓（不阻断记账） |
| 周期 | 单周回撤 5% → 空仓 2 天；总回撤 10% → 停手 1 周 | 顶栏熔断提示 + 复盘入口 |

### 6.2 绝对禁止（系统须识别并警告）

- 逆势抄底（退潮期买入）
- 扛单死等（浮亏超阈仍 hold）
- 亏损补仓
- 非计划内临时买票

### 6.3 功能需求

| ID | 能力 | 状态 |
|----|------|------|
| K-01 | 用户设定「总资金」与风控参数 | **已有** | 持仓区「风控设置」（基础 / 高级两页） |
| K-02 | 单笔风险计算器：成本价 + 止损 % → 建议股数 | **已有** | 登记对话框 + AI 工具 |
| K-03 | 当日已实现 + 浮亏汇总 | **已有** | 风控设置（登记卖出可查看明细）+ 持仓 stats |

## 7. 复盘与纪律

### 7.1 日复盘闭环（30 分钟）

| 步骤 | 内容 | zak 支撑 |
|------|------|----------|
| 市场复盘 | 涨跌停、连板高度、题材梯队、次日情绪预期 | 市场页 + 情绪周期 T-03 + AI 摘要 |
| 交易复盘 | 每笔：买入理由、买卖点、盈亏原因；区分模式内亏损 vs 违规 | 笔记流水 + `trade_journal`（**已有**）；笔记可右键导入流水；误记在 **复盘 → 流水明细** 或笔记中心 **交易流水** Tab 修正 |
| 次日计划 | 总仓位、3–5 只观察股、预设买卖条件 | `trading_plans` + 笔记中心「计划」Tab（**已有**） |

### 7.2 交易计划（新实体）

```text
TradingPlan (date)
├── emotion_stage_expected   # 预期情绪阶段
├── max_position_pct         # 计划总仓位
├── watchlist: [vt_symbol]   # 3–5 只
├── per_symbol:
│   ├── allowed_modes        # 打板 | 半路 | 低吸
│   ├── entry_conditions     # 结构化条件
│   └── exit_conditions
└── notes
```
